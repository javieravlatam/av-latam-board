# AV LATAM BOARD
## Release Checklist v1.0

Completar antes de cada publicación relevante.

---

## A. Datos

- [ ] Fuente oficial identificada.
- [ ] Pipeline ejecutado.
- [ ] Totales conciliados.
- [ ] No existen hardcodeos nuevos.
- [ ] Faltantes mostrados como pendientes.
- [ ] Fecha de actualización visible.

## B. Integridad

- [ ] Mismas cifras en todos los módulos.
- [ ] Presupuesto coincide.
- [ ] Venta real coincide.
- [ ] IEC coincide.
- [ ] Cumplimiento coincide.
- [ ] SIC consume Board.
- [ ] PDF coincide con dashboard.

## C. Funcionalidad

- [ ] Botones funcionan.
- [ ] Filtros funcionan.
- [ ] Selectores actualizan gráficos.
- [ ] Tabs funcionan.
- [ ] Navegación funciona.
- [ ] Exportación funciona.
- [ ] No hay errores de consola.

## D. UX y Formato Javier

- [ ] Paleta correcta.
- [ ] Tipografía legible.
- [ ] Sin saturación.
- [ ] KPIs no duplicados.
- [ ] Jerarquía clara.
- [ ] Responsive validado.
- [ ] Sin textos pequeños.

## E. Código

- [ ] Sin archivos temporales.
- [ ] Sin node_modules.
- [ ] Sin locks.
- [ ] Pruebas aprobadas.
- [ ] CHANGELOG actualizado.
- [ ] README actualizado si aplica.

## F. Publicación

- [ ] Commit correcto.
- [ ] Push realizado.
- [ ] GitHub Pages actualizado.
- [ ] URL pública verificada.
- [ ] Último commit coincide con producción.
- [ ] Caché descartada con evidencia si hubo problema.

## G. Aprobación

- [ ] Validación técnica.
- [ ] Validación funcional.
- [ ] Validación de datos.
- [ ] Aprobación de Gerencia General cuando corresponda.
